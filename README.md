# Project Documentation

## Overview
This project contains multiple pages including a landing page and a profile page, each with their own structure and styling.

## Project Structure

### LandingPage
- **Purpose**: Main entry point for the application
- **Location**: `LandingPage/` folder
- **Contents**: 
  - `index.html` - Landing page HTML
  - `css/styles.css` - Landing page styles
  - `assets/` - Media files
- **Details**: See [LandingPage/README.md](LandingPage/README.md) for more information

### ProfilePage
- **Purpose**: User profile display page
- **Location**: `ProfilePage/` folder
- **Contents**:
  - `index.html` - Profile page HTML
  - `styles.css` - Profile page styles
- **Details**: See [ProfilePage/README.md](ProfilePage/README.md) for more information

## Getting Started

1. Navigate to either `LandingPage` or `ProfilePage` folder
2. Open the respective `index.html` file in a web browser
3. Refer to individual README files for folder-specific details

## Development

- **HTML**: Update HTML files to modify page content and structure
- **CSS**: Edit CSS files to customize styling
- **Assets**: Add images and media files to the `assets/` folder where applicable

## Browser Compatibility
- Test across modern browsers (Chrome, Firefox, Safari, Edge)
- Ensure responsive design works on mobile devices

## Contributing
- Keep each page's CSS separate and organized
- Document any new features or changes in respective README files
- Maintain consistent naming conventions for assets

## Support
For specific details about each page, please refer to their individual README files.
